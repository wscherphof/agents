# Design: extract merged agent instructions into an imported file

## Problem

`merge_claude_md` in
`.claude/hooks/session-start/scripts/merge-agent-settings.sh` injects the
project repo's mirrored `CLAUDE.md` content *inline* into the root `CLAUDE.md`,
between `<!-- BEGIN MERGED AGENT INSTRUCTIONS -->` and
`<!-- END MERGED AGENT INSTRUCTIONS -->` markers. To do this it must: read the
existing root `CLAUDE.md`, strip the old block with an embedded Python snippet,
re-assemble base + markers + body, and rewrite the file. This couples the merge
script to the root `CLAUDE.md` layout and leaves auto-generated content sitting
in an otherwise hand-maintained file.

## Goal

Write the mirrored instructions to their own file,
`.claude/merged-agent-instructions.md`, and `@import` that file from the root
`CLAUDE.md`. The root `CLAUDE.md` becomes 100% hand-maintained; the merge script
sheds its marker/strip machinery and simply writes one generated file.

## Design

### 1. Generated file: `.claude/merged-agent-instructions.md`

- Auto-generated by `merge_claude_md`. Tracked and pushed like the other
  mirrored outputs (only `/src/` and `*.log` are gitignored).
- Begins with a generated-file marker comment:
  `<!-- auto-generated by merge-agent-settings.sh; do not edit -->`
- Followed by the same per-layer sections the managed block produces today:
  `## From <account>/<repo> (<label>)` headers, each followed by that layer's
  `CLAUDE.md` content.
- A committed **placeholder** (the marker comment only, no body) ships on the
  `main` template branch, so the `@import` resolves on `main` and on
  freshly-created branches before the merge script has run. This mirrors the
  `conf/CLAUDE.md` empty-template pattern.

### 2. Root `CLAUDE.md` (hand-maintained)

- Remove the `<!-- BEGIN MERGED AGENT INSTRUCTIONS (auto-generated, do not
  edit) -->` / `<!-- END MERGED AGENT INSTRUCTIONS -->` block.
- Append, at the end of the file (where the merged content used to live):

  ```markdown
  The project repo's own agent instructions, mirrored in by the session-start
  hook, are imported here:

  @.claude/merged-agent-instructions.md
  ```

- The root `CLAUDE.md` now has two static imports: `@conf/CLAUDE.md` (manual
  overlay, near the top) and `@.claude/merged-agent-instructions.md` (generated
  project instructions, at the bottom). The merge script never reads or writes
  the root `CLAUDE.md`.

### 3. `merge_claude_md` rewrite

Remove:
- The `CLAUDE_BEGIN` / `CLAUDE_END` constants.
- The embedded Python that strips the old managed block from the existing
  `CLAUDE.md`.
- All base-preservation / append / "remove CLAUDE.md when empty" logic tied to
  the root file.

Keep / change:
- The per-layer body assembly loop is unchanged (same `## From ...` headers,
  same skip-empty-layer behavior).
- Write `marker-comment + body` to `$DEST/.claude/merged-agent-instructions.md`.
- When the body is empty (no source layer has a `CLAUDE.md`), write the
  marker-comment-only placeholder. **Never delete the file** — deleting it would
  leave the root `CLAUDE.md`'s `@import` dangling.

### 4. Documentation / comment updates

- `conf/CLAUDE.md` line ~11: change the "`MERGED AGENT INSTRUCTIONS` block of
  the root CLAUDE.md" reference to "`.claude/merged-agent-instructions.md`,
  imported by the root CLAUDE.md".
- merge script section-7 header comment: describe writing the separate file
  instead of the managed block.
- `README.md` step 5: describe the mirror writing
  `.claude/merged-agent-instructions.md` (imported by the root `CLAUDE.md`)
  rather than injecting into `CLAUDE.md`.

## Out of scope / accepted trade-offs

- **Migration of existing branches.** Because the merge script no longer
  rewrites the root `CLAUDE.md`, existing project settings branches keep their
  old inline managed block until they pick up `main`'s new `CLAUDE.md` (the same
  way any scaffolding change propagates). We deliberately do **not** add
  one-time "strip the legacy block" cleanup to the merge script — keeping the
  script clean is the goal of this change.
- No change to the other mirror functions (`mirror_settings`, `mirror_mcp`,
  `copy_referenced_files`, `mirror_dir`).

## Verification

1. Run a reproduction of the new `merge_claude_md` against sample source layers;
   confirm `.claude/merged-agent-instructions.md` is written with the marker
   comment + `## From ...` sections, and that the root `CLAUDE.md` is untouched.
2. With no source `CLAUDE.md` present, confirm the placeholder (marker comment
   only) is written and the file is not deleted.
3. Confirm the root `CLAUDE.md` contains a bare `@.claude/merged-agent-instructions.md`
   import line on its own line.
4. Confirm no remaining references to `BEGIN MERGED AGENT INSTRUCTIONS` /
   `CLAUDE_BEGIN` / `CLAUDE_END` outside the docs/superpowers history.
